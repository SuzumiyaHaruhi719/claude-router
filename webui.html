    Virtual Models

    A virtual model is a name Claude Code can request that routes per-request by condition (image / web-search / long-context / keyword), or that you flip by hand between a list of candidate backends. No fusion tool-loop — one backend answers each turn. Virtual models resolve before routes. Add a router model, set ANTHROPIC_MODEL=router, and switch backends with one click — no restart.

    Dummy Key
    
        
    ANTHROPIC_API_KEY=claude-router is intentionally non-empty so Claude Code will send requests. The router ignores that dummy key and injects the selected backend's OAuth token, x-api-key, or bearer token upstream.
    
      
    
    
    
    
    
      
    
        
        
    Set ANTHROPIC_BASE_URL to this router and ANTHROPIC_API_KEY=claude-router. Real upstream credentials never leave the backend config.
